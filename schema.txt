# 🗄️ Lazur — PostgreSQL Schema (Source of Truth)

> Principles
> 
- App is untrusted
- Server enforces limits
- Usage is auditable
- Billing is safe
- Schema supports upgrades without rewrites

---

## 1️⃣ USERS

```sql
CREATE TABLE users (
  id UUIDPRIMARY KEYDEFAULT gen_random_uuid(),
  email TEXTNOT NULLUNIQUE,
  auth_provider TEXTNOT NULL,-- google | github | email
  created_at TIMESTAMPTZNOT NULLDEFAULT now()
);

```

✅ Owned by **website**

✅ Referenced everywhere else

---

## 2️⃣ PLANS

```sql
CREATE TABLE plans (
  id UUIDPRIMARY KEYDEFAULT gen_random_uuid(),
  name TEXTNOT NULLUNIQUE,-- free | pro | team
  monthly_limit_secondsINTEGERNOT NULL,
  monthly_limit_requestsINTEGERNOT NULL,
  price_centsINTEGERNOT NULL,
  created_at TIMESTAMPTZNOT NULLDEFAULT now()
);

```

✅ Static table

✅ Seed once

✅ Used for enforcement

---

## 3️⃣ USER PLANS (SUBSCRIPTIONS)

```sql
CREATE TABLE user_plans (
  id UUIDPRIMARY KEYDEFAULT gen_random_uuid(),
  user_id UUIDNOT NULLREFERENCES users(id)ONDELETE CASCADE,
  plan_id UUIDNOT NULLREFERENCES plans(id),
  status TEXTNOT NULL,-- active | canceled | past_due
  stripe_customer_id TEXT,
  stripe_subscription_id TEXT,
  started_at TIMESTAMPTZNOT NULLDEFAULT now(),
  ends_at TIMESTAMPTZ
);

CREATE INDEX idx_user_plans_user_idON user_plans(user_id);

```

✅ Updated via **Stripe webhooks**

✅ App only reads this

---

## 4️⃣ DEVICES

Each desktop install = one device.

```sql
CREATE TABLE devices (
  id UUIDPRIMARY KEYDEFAULT gen_random_uuid(),
  user_id UUIDNOT NULLREFERENCES users(id)ONDELETE CASCADE,
  device_fingerprint TEXTNOT NULL,
  os TEXTNOT NULL,-- macos | windows
  app_version TEXTNOT NULL,
  last_seen TIMESTAMPTZ,
  created_at TIMESTAMPTZNOT NULLDEFAULT now(),
UNIQUE (user_id, device_fingerprint)
);

CREATE INDEX idx_devices_user_idON devices(user_id);

```

📌 `device_fingerprint` = hashed UUID generated by app

📌 Prevents unlimited device abuse

---

## 5️⃣ DESKTOP AUTH TOKENS (ONE-TIME)

Used for **browser → app deep-link auth**

```sql
CREATE TABLE desktop_auth_tokens (
  id UUIDPRIMARY KEYDEFAULT gen_random_uuid(),
  user_id UUIDNOT NULLREFERENCES users(id)ONDELETE CASCADE,
  token TEXTNOT NULLUNIQUE,
  expires_at TIMESTAMPTZNOT NULL,
  used_at TIMESTAMPTZ
);

CREATE INDEX idx_desktop_tokens_tokenON desktop_auth_tokens(token);

```

Rules:

- TTL: 2–5 minutes
- One-time use
- Deleted or marked used after verification

---

## 6️⃣ APP SESSIONS (DEVICE-BOUND)

```sql
CREATE TABLE app_sessions (
  id UUIDPRIMARY KEYDEFAULT gen_random_uuid(),
  user_id UUIDNOT NULLREFERENCES users(id)ONDELETE CASCADE,
  device_id UUIDNOT NULLREFERENCES devices(id)ONDELETE CASCADE,
  session_token TEXTNOT NULLUNIQUE,
  expires_at TIMESTAMPTZ,
  created_at TIMESTAMPTZNOT NULLDEFAULT now()
);

CREATE INDEX idx_app_sessions_user_idON app_sessions(user_id);

```

📌 This is what the **desktop app uses**

📌 Rotatable later

---

## 7️⃣ USAGE — DAILY AGGREGATION (IMPORTANT)

Never store per-keystroke data.

```sql
CREATE TABLE usage_daily (
  id UUIDPRIMARY KEYDEFAULT gen_random_uuid(),
  user_id UUIDNOT NULLREFERENCES users(id)ONDELETE CASCADE,
  device_id UUIDNOT NULLREFERENCES devices(id)ONDELETE CASCADE,
  usage_dateDATENOT NULL,
  seconds_usedINTEGERNOT NULLDEFAULT0,
  requestsINTEGERNOT NULLDEFAULT0,
  created_at TIMESTAMPTZNOT NULLDEFAULT now(),
UNIQUE (user_id, device_id, usage_date)
);

CREATE INDEX idx_usage_user_dateON usage_daily(user_id, usage_date);

```

✅ Billing-safe

✅ Easy to query

✅ Easy to reset monthly
